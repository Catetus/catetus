# Shared QAT-PLY codec test fixtures

This directory contains the reference encoder + a synthetic fixture builder
that all three plugin test suites (`threejs-qat`, `babylonjs-qat`,
`cesium-qat`) consume. Keeping it in a single source means the three decoders
are asserted against the same bytes — bit-exact parity is the goal.

## Files

- `buildFixture.ts` — pure-TypeScript reference encoder that produces a
  byte-identical QAT-PLY to what the Python `save_ply` path emits.
  Exports `buildSyntheticFixture(N)` which returns the bytes + the
  ground-truth dequantized arrays for every codec path.
- `bonsai-tiny.qat.ply` — a tiny pre-built natural fixture (256 anchors,
  ~9 KB) covering the constant-strip + fp32 SH-DC paths. Generated
  in-process by the reference encoder (deterministic LCG seed).
- `generate-bonsai-tiny.mjs` — Node script that regenerates
  `bonsai-tiny.qat.ply` from the reference encoder. Run with
  `node generate-bonsai-tiny.mjs` to re-emit the file.

## Why synthetic instead of a real Inria bonsai

A 5-MB bonsai sample would balloon git LFS for a public reference
implementation that's already small. The synthetic fixture is:

- deterministic (LCG seed `0xdeadbeef`)
- complete (touches every codec path: constants, int8, int4)
- inspectable (the encoder source is right next to the decoder)
- bit-exact reproducible across node/browser/CI

Real-world QAT PLYs (bonsai, bicycle, etc.) generated by the deployed
`/qat-3dgs` Modal endpoint render correctly in the example HTML pages —
point the `?src=` query param at one to verify.
