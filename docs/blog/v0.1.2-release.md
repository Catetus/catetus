# SplatForge v0.1.2 — production pipeline for Gaussian Splats

**Status:** draft (2026-05-15). Replace `TODO`s before publishing.

---

SplatForge is the production-optimization pipeline for Gaussian Splats — the FFmpeg / Cloudinary layer between training tools (which spit out 100–900 MB raw PLY) and runtimes (which need byte-deterministic, fidelity-validated, standards-aligned assets).

v0.1.2 ships the surface that lets you actually use it:

1. **SplatForge Cloud is live** at [`https://splatforge-api.fly.dev`](https://splatforge-api.fly.dev). Free-tier `POST /v1/jobs` accepts a `.ply` and returns a `.glb` plus a JSON fidelity report. Paid-tier `POST /v1/jobs/:id/repack` dispatches to A100 differentiable-repack at ~$0.05/scene with **+6.4 dB PSNR over opacity-prune at 50% byte budget**.
2. **`splatforge submit` CLI** — `splatforge submit scene.ply --preset web-mobile` proxies bytes to Cloud and prints the public output URL. Live-tested in this release against the deployed Fly API.
3. **Public SplatBench leaderboard** at `splatforge.dev/bench` with per-scene compression × fidelity. Now lights up a **splat-transform comparison column** (PlayCanvas's npm CLI, the obvious "why not just use this?") — SplatForge v0.1.2 median is **21.88× vs splat-transform v2.1.1's 5.91×** on the same 13-scene synthetic corpus.
4. **KHR_gaussian_splatting reference implementation + conformance test suite** — 23 normative clauses, 10 golden fixtures, CI workflow. We're submitting this to Khronos as the public reference impl before the spec ratifies in Q2 2026.
5. **OpenUSD `ParticleField3DGaussianSplat`** — USDC binary writer round-trips **bit-exact-as-USDA against Apple `usdcat` 0.25.2** on three reference scenes. 10 spec gaps captured for the OpenUSD WG.
6. **Cesium 3D Tiles preset** — `splatforge optimize --preset geospatial` emits a `tileset.json` + 4-level LOD pyramid that drops directly into Cesium ion.
7. **WebGPU compute decode + GPU radix sort** — 1 M splats at 127 fps on M-series; streaming-tile viewer adapter for the 30 GB / 60 fps mobile target.
8. **GitHub Action** (`splatforge/optimize-action`) — drop in any repo, get a PR fidelity badge that fails CI on regression.

We also closed the loop on three pieces of research that the queue had as "promising but unvalidated":

- **PostHAC** lossless entropy coding stacks to **34.5× on bonsai** (1.57× over 22× quant baseline, net-residual ratio improves with scene size).
- **SwVQ** saliency-weighted vector quantization beats uniform 8-bit by **37% on the deterministic synthetic** (target was 17%).
- **Point-Cloud Codec Adapter** (Draco) adds **1.80× orthogonal compression** on bonsai at 0 dB ΔPSNR, composable with PostHAC.

And on three more that we **killed cleanly** rather than wasting another quarter on:

- **F-3DGS factorized coordinates** — paper claimed 90% size reduction; on a real corpus, positions alone (where F-3DGS applies) are ~10-15% of file bytes, so the realistic whole-file savings is ~2.7%. Headline doesn't survive engineering.
- **Splat-Δ sparse anchors** — every re-attack lever (stride sweep, attribute selection, learned MLP) returned ≤1.0× orthogonal lift over SwVQ × PostHAC. PostHAC's hash-grid context already extracts the smooth-field signal Splat-Δ targets.
- **CodecGS-Lite hardware video path** — published 146× headline on bicycle measured at 20.3× on our 4090 (7× shortfall). Engineering-realistic ceiling ~30-60× with RDO + finetune + 10-bit channel routing. Deferred to Q5; do not headline 146× anywhere.

The ship/kill discipline is a feature, not a side effect: every result in the SplatBench corpus is reproducible from a public corpus ID, every kill decision is in `research/queue.md` with the test that produced it, and every shipped preset is byte-deterministic given `(input, version, seed)`.

## Why this matters

PlayCanvas `splat-transform` is a good Gaussian-Splat conversion CLI. It is the obvious answer for one-off conversions inside the PlayCanvas / SOG ecosystem. **It is not a production pipeline.** It produces no fidelity report, no deterministic outputs, no hosted API, no PR badge for CI, no MCP integration, no standards conformance suite. Production splat assets — the kind that ship to a million users on Safari, or render inside an Omniverse pipeline, or live behind a Bentley/Cesium geospatial layer — need every one of those.

SplatForge v0.1.2 is the bet that the production surface matters more than the conversion algorithm. The 3.7× compression-ratio lead on our own corpus is the by-product of the algorithms we run; the durable advantage is the rest of the stack.

## Try it

```bash
# Free tier on Cloud (proxy-upload your local PLY, get a public .glb back)
brew install splatforge   # TODO: replace once the homebrew tap lands
splatforge submit your.ply --preset web-mobile
# → https://splatforge-api.fly.dev/...

# Local (no key, no network, runs deterministically)
splatforge optimize --preset web-mobile your.ply --output your.glb
splatforge fidelity your.glb --baseline your.ply

# Validate against the spec
splatforge spec-check your.glb
```

GitHub Action:

```yaml
- uses: splatforge/optimize-action@v1
  with:
    api-key: ${{ secrets.SPLATFORGE_API_KEY }}
    preset: web-mobile
```

The full SplatBench corpus, reproducer scripts, and submission-ready conformance suite are in the [public repo](https://github.com/montabano1/SplatForge). The leaderboard at `splatforge.dev/bench` lets you compare any encoder against the same scenes — submit yours via the form at the bottom of the page.
